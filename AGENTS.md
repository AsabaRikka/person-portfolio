# 项目概述

- 本项目是一个个人作品集网站，目标是快速展示「我是谁、擅长什么、做过哪些代表项目、如何联系我」。
- 主要页面/区块包括：首页（大标题+简介）、关于我、项目展示、联系方式。
- 技术栈：React + TypeScript + Vite + Tailwind CSS，数据以本地静态配置为主，少量使用 localStorage 做体验增强。

# 开发规范

- 使用 React 函数组件与 Hooks，配合 TypeScript，组件 Props 需要定义类型。
- 代码结构遵循 `TECH_DISIGN.md` 中的精简目录：组件统一放在 `src/components/`，静态数据放在 `src/data/`。
- 代码中适当添加注释，尤其是：
  - 关键逻辑（如滚动锚点跳转、滚动进入视口触发动画、localStorage 读写封装）。
  - 容易引起误解的样式或交互处理。
  - 任何与 PRD/技术设计不完全一致、需要特殊处理的地方。
- 保持组件职责单一：Section 组件负责页面区块布局，基础组件负责可复用 UI 单元（如项目卡片）。
- 命名规范：
  - 组件/文件：PascalCase（如 `HeroSection.tsx`）。
  - 变量/函数：camelCase。
  - Tailwind 类名尽量保持语义清晰、分组合理（布局类在前、颜色和动效类在后）。

# 设计要求

- 确保移动端适配优先（Mobile First），在常见手机宽度（375–430px）下有良好体验：
  - 布局在小屏幕上采用单列为主，文字字号和行距适合阅读。
  - 导航在移动端使用汉堡菜单或简单下拉，不要占用过多纵向空间。
  - 按钮和卡片点击区域需足够大，便于手指点击。
- 响应式实现：
  - 使用 Tailwind 的响应式断点类（如 `sm:`, `md:`, `lg:`）控制不同屏幕布局。
  - 桌面端可以拉宽内容区并增加左右留白，但不要影响移动端的阅读体验。
- 动效：
  - 使用轻量级的淡入、上滑、Hover 动效，增强页面气质，但不要喧宾夺主。
  - 动画优先使用 `transform`、`opacity` 等属性，以 Tailwind 工具类实现，避免复杂的自定义动画逻辑。

# 注意事项（性能与体验优化）

- 首屏性能：
  - 控制首页资源体积，图片资源做压缩和合理尺寸裁剪，避免超大原图直接加载。
  - 对非首屏必要的图片（如部分项目截图）可考虑懒加载（逐步迭代）。
- 渲染性能：
  - 动画尽量使用 CSS 过渡，不要在滚动事件中频繁执行重计算。
  - 如果使用 `IntersectionObserver` 做滚动进入动画，要注意解绑与复用，避免创建过多观察器。
- localStorage 使用：
  - 通过统一封装的 Hook（如 `useLocalStorage`）读写，处理好 JSON 序列化和默认值。
  - 不在渲染过程中频繁写入 localStorage，读写应尽量放在副作用中（`useEffect`）。
- 可维护性：
  - 文案和项目数据放在 `data` 文件中集中管理，避免散落在多个组件里，方便后期更新。
  - 新增组件或重构目录时，注意同步更新 `TECH_DISIGN.md`，保持设计文档与实现基本一致。

个人作品集网站 PRD（MVP 版）

## 一、产品目标
在 1 分钟内展示清楚：你是谁、擅长什么、与你目标岗位的匹配度，以及如何联系到你。

通过「身份与简介、技能、工作经历、教育经历、代表项目（可选科研经历）」快速建立信任并促成进一步沟通。

## 二、核心页面与功能

### 2.1 首页（Home）
- **内容**：
  - 大标题：一句话身份定位（如「前端开发工程师」）。
  - 简短简介：1–2 句说明方向/优势。
  - 快速按钮：跳转到「工作经历」「项目」「联系」区域。
- **MVP 要求**：静态内容即可，无复杂交互。

### 2.2 关于我（About）
- **内容**：
  - 基本信息：姓名、学历层次、所在地（年龄可选）。
  - 头像/照片（可选）。
  - 求职意向：目标岗位/方向、期望地点（可选）、到岗时间（可选）。
  - 技能简表：列出主要技术栈或能力标签。
  - 简略经历：教育/工作/实习，只保留最重要的 2–3 条。
- **MVP 要求**：单页区块呈现，偏类似简历摘要。

### 2.3 工作经历（Experience）
- **内容**：按「一段经历 = 一个大块」展示，每段经历包含：
  - **经历头部（大块标题行）**：公司名称｜岗位名称｜起止时间
  - **工作亮点卡片列表**（每段经历下可有多张亮点卡片）：
    - 标题行：一句话概括亮点（解决了什么问题/带来什么价值）
    - 亮点要点（bullet list）：2–4 条（每条 1 句话，强调“做了什么、带来什么价值”）
    - 能力标签（可选）：如 React、性能优化、工程化、协作等
- **MVP 要求**：
  - 至少 2 段工作经历（大块）。
  - 每段经历至少 4 张亮点卡片。
  - 仅做列表展示；详情页/展开页后续迭代。

### 2.4 项目展示（Projects）
- **内容**：项目卡片列表。
  - 每个项目卡片包含：
    - 项目名称
    - 一句话描述（做什么/解决什么问题）
    - 技术栈标签（如 React、Node.js 等）
    - 可选链接：在线 Demo、GitHub 源码
- **MVP 要求**：
  - 至少 3 个代表性项目。
  - 仅做列表 + 外链；项目详情页后续迭代。

### 2.5 教育经历（Education）
- **内容**：教育经历卡片列表，从本科开始（或按最高学历优先），展示学校与学习信息。
  - 每个经历卡片包含：
    - 标题行：学校名称｜专业名称｜学历层次｜学习起止时间
    - 亮点要点（可选）：1–3 条（课程/竞赛/科研/项目等，一句话即可）
    - 荣誉/成果（可选）：如奖学金、竞赛奖项、论文/专利等
- **MVP 要求**：
  - 仅做列表展示；详情页/展开页后续迭代。

### 2.6 联系方式（Contact）
- **内容**：清晰可点击/可复制的联系方式（可配图标）。
  - 邮箱
  - GitHub 链接
  - 其他社交/博客链接（至少 1 个，如小红书等）
  - 电话（可选）
  - 微信（可选，建议以二维码图片形式展示）
- **MVP 要求**：保证在移动端易点击、易复制。

## 三、设计与交互要求（MVP 范围）

### 3.1 响应式设计
- 适配手机与桌面端：
  - 手机：单列布局，文字字号和间距适合阅读。
  - 桌面：可采用两列或更宽布局。
- 导航使用锚点跳转到页面内部区块（Home / About / Experience / Projects / Education / Contact）。

### 3.2 基础动效
- 动效风格：简洁、轻量，不影响加载速度。
- MVP 动效范围：
  - 页面加载时首页标题和简介淡入。
  - 滚动进入可视区域时，卡片类模块轻微淡入/上滑（如 Projects/Experience/Education）。
  - 按钮与卡片 Hover 态有基础过渡（颜色/阴影/位移变化）。

## 四、非功能需求（MVP）
- **性能**：
  - 图片在保证清晰度的前提下进行压缩。
  - 控制首屏加载资源体积，优先加载首页必要内容。
- **基础 SEO**：
  - 设置站点标题（包含角色，如「个人作品集 · 前端开发」）。
  - 配置描述（meta description），简要说明个人身份与主要技能。

## 五、可迭代项
- 5.1 网页信息导出「简历模板 PDF」
  - **背景与目标**：
    - 访客或本人希望将网页信息一键导出为可投递/可打印的简历 PDF。
    - 导出内容与网页保持一致，避免维护两份简历。
  - **范围（V1）**：
    - 导出内容（数据源统一来自 `src/data/*`）：
      - 基本信息：姓名、title、location
      - 求职意向：目标方向/地点（可选）/到岗时间（可选）
      - 技能标签
      - 工作经历：经历大块（公司/岗位/时间）+ 亮点卡片列表（标题/要点/标签可选）
      - 项目展示：按 `order` 导出（可先导出全部，后续可加“仅导出前 N 个”）
      - 教育经历：学校/专业/学历/学习时间 + 可选要点/荣誉成果
      - 联系方式：邮箱、GitHub、其他链接（电话/微信可选）
    - V1 不做：
      - 多模板切换、多语言版本、在线编辑
  - **入口与交互（V1）**：
    - 入口位置（可选其一，后续可同时提供）：Hero 按钮 / Navbar 菜单 / Contact 区块按钮
    - 点击入口 → 打开「简历预览页/打印视图」
    - 预览页提供：下载/打印 PDF、返回主页
    - 异常提示：生成失败时给出提示并支持重试
  - **简历模板要求（V1）**：
    - 纸张：A4，适合黑白打印
    - 优先一页，内容较多允许自动分页
    - 建议区块顺序：
      1) Header：姓名 + title + location + 联系方式
      2) 求职意向（可选）
      3) 技能
      4) 工作经历
      5) 项目展示
      6) 教育经历
    - 链接应可点击（PDF 内保留链接）
  - **数据规则与容错**：
    - 列表排序：Experience/Education/Projects 按 `order` 升序
    - 可选字段为空则不展示（电话/微信/荣誉等）
    - 文本长度建议：每张亮点卡 bullets 最多 2–4 条（与 PRD 2.3 一致）
  - **验收标准（V1）**：
    - 能从入口生成/下载（或打印保存）PDF
    - PDF 内容与网页一致（以 `src/data` 为准）
    - 可选字段缺失不会出现空白占位或布局错乱
    - 工作经历按“经历大块 + 亮点卡片”结构正确导出
  - **实现方式备注（供技术设计对齐）**：
    - 优先方案：提供专用简历页面，使用浏览器打印 `window.print()` 让用户“另存为 PDF”
    - 后续可选：引入前端 PDF 生成库，做真正的一键下载

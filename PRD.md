个人作品集网站 PRD（MVP 版）

## 一、产品目标
展示个人身份、技能与代表项目，让访客在 1 分钟内清楚了解「你是谁、擅长什么、可联系到你」。

## 二、核心页面与功能

### 2.1 首页（Home）
- **内容**：
  - 大标题：一句话身份定位（如「前端开发工程师」）。
  - 简短简介：1–2 句说明方向/优势。
  - 快速按钮：跳转到「项目」和「联系」区域。
- **MVP 要求**：静态内容即可，无复杂交互。

### 2.2 关于我（About）
- **内容**：
  - 基本信息：姓名、角色（职位）、所在地（可选）。
  - 技能简表：列出主要技术栈或能力标签。
  - 简略经历：教育/工作/实习，只保留最重要的 2–3 条。
- **MVP 要求**：单页区块呈现，偏类似简历摘要。

### 2.3 项目展示（Projects）
- **内容**：项目卡片列表。
  - 每个项目卡片包含：
    - 项目名称
    - 一句话描述（做什么/解决什么问题）
    - 技术栈标签（如 React、Node.js 等）
    - 可选链接：在线 Demo、GitHub 源码
- **MVP 要求**：
  - 至少 3 个代表性项目。
  - 先只做列表 + 外链，项目详情页可后续迭代。

### 2.4 联系方式（Contact）
- **内容**：
  - 邮箱（mailto 链接）
  - GitHub 链接
  - 至少 1 个其他社交媒体或博客链接（如 LinkedIn / 掘金 / 知乎等）
- **MVP 要求**：保证在移动端易点击、易复制。

## 三、设计与交互要求（MVP 范围）

### 3.1 响应式设计
- 适配手机与桌面端：
  - 手机：单列布局，文字字号和间距适合阅读。
  - 桌面：可采用两列或更宽布局。
- 导航使用锚点跳转到页面内部区块（Home / About / Projects / Contact）。

### 3.2 基础动效
- 动效风格：简洁、轻量，不影响加载速度。
- MVP 动效范围：
  - 页面加载时首页标题和简介淡入。
  - 滚动进入可视区域时，项目卡片轻微淡入/上滑。
  - 按钮和项目卡片 Hover 态有基础过渡（颜色/阴影变化）。

## 四、非功能需求（MVP）
- **性能**：
  - 图片在保证清晰度的前提下进行压缩。
  - 控制首屏加载资源体积，优先加载首页必要内容。
- **基础 SEO**：
  - 设置站点标题（包含角色，如「个人作品集 · 前端开发」）。
  - 配置描述（meta description），简要说明个人身份与主要技能。

## 五、本期不做（可迭代项）
- 多语言切换。
- 博客/文章模块。
- 项目详情页的深入案例分析。
